# HybridMC 로드맵

> 자바 에디션과 베드락 에디션을 **동시에 네이티브로** 지원하고 플러그인까지 지원하는,
> Minecraft 서버 코드를 일절 사용하지 않는 **클린룸** 서버 구동기.

> 📋 이 문서는 **전략/개요**다. 마일스톤을 손에 잡히는 작은 작업 단위(육하원칙)로 쪼갠
> **상세 작업 분해는 [TASKS.md](TASKS.md)** 참고.

## 확정된 핵심 결정

| 축 | 결정 |
|---|---|
| 크로스플레이 | **네이티브 듀얼 스택** — 단일 통합 코어 위에 자바·베드락 프로토콜을 둘 다 네이티브 구현 |
| 타겟 버전 | **자바·베드락 26.2 단일 고정** (다버전 추상화 없음) |
| 플러그인 API | **독자 API 신규 설계** (Bukkit/Spigot 호환 아님) |
| MVP 범위 | **바닐라 서바이벌 파리티**(최종 DoD) — 수직 슬라이스 마일스톤으로 분할 |

> ⚠️ "네이티브 듀얼 스택 + 바닐라 파리티"는 모든 축에서 최고 난이도다. 단일 메이저 버전 기준으로도
> 수 년 규모의 작업이며, "바닐라 파리티"는 첫 출시물이 아니라 **도착점**으로 둔다.

## 엔지니어링 스파인: canonical 레지스트리 + 에디션 매핑

자바와 베드락은 같은 게임처럼 보여도 와이어 레벨에서 거의 다른 게임이다.

| 항목 | 자바 26.2 | 베드락 26.2 |
|---|---|---|
| 전송 | Netty **TCP**, VarInt 프레이밍, zlib 압축, AES 암호화 | **RakNet/UDP**, 패킷 batch, 별도 암호화 체인 |
| 인증 | Mojang/Microsoft (online mode) | Xbox Live 로그인 체인 |
| 블록 식별 | paletted **global state ID** | name + states → **runtime ID** |
| 청크 | 섹션(16³) + 글로벌 팔레트 | subchunk + 로컬 팔레트 |
| 엔티티 메타 | 자바 metadata 인덱스 | 베드락 actor data 키 |
| UI | 채팅/명령 중심 | Form(네이티브 UI), 다른 인벤토리 윈도우 |

→ 모든 것을 **단일 진실원본(통합 코어)** 으로 수렴시키고, 프론트엔드에서 각 에디션으로 직렬화한다.
매 마일스톤은 **"통합 모델 구현 → 자바 직렬화 → 베드락 직렬화"** 3겹으로 진행한다.

> **클린룸 원칙**: Mojang 서버 코드/디컴파일 금지. 블록·아이템·엔티티 정의와 매핑 데이터는
> 공개 데이터셋(예: `minecraft-data`(PrismarineJS), Geyser/Cloudburst 매핑 테이블, 공개 data report)에서
> 소싱해 **빌드 타임 파이프라인**으로 레지스트리를 생성한다. 코드가 아니라 데이터를 가공한다.

## 기술 스택

- **언어/런타임**: Kotlin/JVM, JDK 21 toolchain
- **네트워크**: 자바 = Netty(TCP) · 베드락 = RakNet(Netty UDP 위) — 둘 다 Netty 파이프라인으로 통일
- **동시성**: 코루틴 기반 IO. **게임 틱은 초기엔 결정론적 단일 스레드 + 비동기 IO**.
  영역기반 멀티스레딩은 나중(M8).
- **직렬화/저장**: 자체 NBT 구현, 월드 저장은 자체 포맷 권장(Anvil 호환은 실익 적음)
- **산출물**: `:app` 모듈의 단일 실행 jar = 구동기

## 모듈 구조

```
:core             수학(vec/AABB/chunk좌표), NBT, 버퍼/VarInt 코덱, 이벤트버스, 스케줄러
:registry         canonical 블록스테이트/아이템/엔티티 레지스트리 + 에디션 매핑 + 데이터 파이프라인
:protocol-java    자바 26.2 패킷 코덱 + 상태머신(handshake/status/login/config/play)
:protocol-bedrock 베드락 26.2 RakNet + 패킷 코덱 + 로그인 체인
:network          통합 세션/연결 추상화(PlayerConnection), 두 프론트엔드 바인딩
:world            월드/청크/블록스테이트 모델, 청크 저장(IO), 라이팅
:worldgen         노이즈 지형, 바이옴, 광물, 구조물
:entity           엔티티 컴포넌트 시스템, 메타데이터 동기화, AI/경로탐색
:game             게임룰: 크래프팅/제련/인챈트, 전투, 레드스톤, 유체, 물리
:api              ★ 공개 플러그인 API (인터페이스 전용)
:server           메인 틱 루프, 플레이어 관리, 명령어, 디멘션
:plugin-host      플러그인 로더/클래스로딩/격리/권한/생명주기
:app              실행 진입점(구동기), 설정/콘솔
```

## 마일스톤 (각 단계 = 실제 굴러가는 서버)

### M0 — 기반 (부트 가능한 빈 서버)
`:core` 코덱·NBT·수학·이벤트버스·틱 스케줄러, `:registry` 골격 + 데이터 파이프라인, `:app` 부팅 + 빈 틱 루프.
**DoD:** `./gradlew :app:run` → 서버가 뜨고 20 TPS로 빈 틱이 돈다.

### M1 — 핸드셰이크 & 로그인 (양 에디션)
자바: TCP·상태머신·압축·암호화·오프라인/온라인 인증. 베드락: RakNet·로그인 체인·배치·암호화.
`:network`의 통합 `PlayerConnection`으로 수렴.
**DoD:** 자바·베드락 클라이언트 둘 다 서버 리스트 핑 + 로그인 상태 진입.

### M2 — 최소 인게임 진입 ★크로스플레이 성립 검증★
평면 빈 월드, 청크 직렬화(자바 섹션 / 베드락 subchunk), 스폰, 이동 동기화, 채팅,
canonical 블록스테이트 → 에디션 runtime ID 매핑(최소 블록 셋).
**DoD:** 자바 유저와 베드락 유저가 같은 월드에서 서로를 보고, 움직이고, 채팅한다. ← 프로젝트 가설 증명.

### M3 — 블록/월드 상호작용
블록 설치/파괴, 인벤토리/핫바, 아이템 사용, 블록 엔티티(상자 등), **전체 블록·아이템 매핑 완성**,
청크 영속화(자체 포맷)·로딩/언로딩.
**DoD:** 크리에이티브 수준 건축이 양 에디션에서 동기화.

### M4 — 엔티티 & 전투 기초
엔티티 컴포넌트 시스템, 메타데이터 동기화, 아이템 엔티티 드랍/픽업, 체력/배고픔/데미지, 기본 근접 전투.
**DoD:** 몹 없는 PvP, 아이템 드랍/픽업.

### M5 — 월드 생성
노이즈 지형, 바이옴, 광물, 동굴, 초기 구조물. (바닐라 시드 1:1 재현은 비현실적 → 자체 월드젠 권장)
**DoD:** 탐험 가능한 절차적 월드.

### M6 — 몹 & 생존 루프
몹 스폰 규칙, A* 경로탐색 + goal 기반 AI, 패시브/적대 몹, 낮/밤 사이클,
크래프팅/제련/인챈트/물약, 경험치, 네더/엔드.
**DoD:** 처음부터 끝까지 생존 게임 루프 성립.

### M7 — 시뮬레이션 심화 (= 바닐라 파리티 본진)
레드스톤, 물/용암 유체, 블록 업데이트 순서, 농사/번식, 날씨, 통계/업적. **끝없는 디테일의 롱테일.**
**DoD:** 핵심 바닐라 메커니즘 재현, 양 에디션 동작.

### M8 — 운영·성능·도구
콘솔/명령 시스템, 권한, 백업, 메트릭, 청크 비동기 IO 최적화, 영역기반 멀티스레딩 검토, 설정 핫리로드.

## 병렬 트랙 — 플러그인 API (M2부터 의식, M8 정식 공개)

독자 API이므로 **내부 인터페이스를 처음부터 공개 가능한 형태로** 설계한다. M2~M7 동안 코어를 만들 때마다
`:api`에 깔끔한 인터페이스를 노출하고, M8에서 로더/격리/권한을 붙여 정식화한다.
듀얼 스택이므로 API에서 **에디션 차이를 추상화**한다 — 예: `player.sendForm(...)`은 베드락은 네이티브 Form,
자바는 채팅/책 UI로 폴백.

## 핵심 리스크

1. **에디션 동작 충돌** — 자바/베드락은 전투 쿨다운·레드스톤 틱·물 흐름·인벤토리가 설계상 다르다.
   "양쪽 다 파리티"는 일부 모순 → M3 즈음 **canonical 동작 정책**(통일 기준)을 명문화.
2. **26.2 프로토콜 데이터 확보** — 자바/베드락 각 26.2의 패킷·레지스트리·매핑 공개 소스 확인(없으면 캡처 리버스).
3. **인증** — online mode(Mojang/Xbox)까지 갈지, 초기엔 offline + 자체 인증으로 갈지.
4. **이동/안티치트 검증** — 서버 권위적 이동 검증 범위.
